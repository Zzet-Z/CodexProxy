## Why This Host Uses A Better Architecture Than 129.204.9.74

Compared with the older host, this machine allowed a cleaner design:

1. `nginx` on Ubuntu already had dynamic `stream` support.
2. public DNS for `chatgpt.com` was clean on this machine.
3. there was no existing HTTPS site to migrate first.

That made 3 improvements possible:

- use `nginx stream + ssl_preread` instead of `sniproxy`
- avoid the old `chatgpt.com -> fixed Cloudflare IP` workaround
- isolate the stream front door in a separate `nginx-xproxy` instance running as `xproxy`, so NAT capture does not affect future main-site `nginx` traffic

## Final Architecture

There are 2 nginx instances:

1. main `nginx.service`
   - keeps standard HTTP role
   - currently listens on `80`
   - serves a local HTTPS placeholder on `127.0.0.1:8443`
   - is reserved for future site hosting

2. `nginx-xproxy.service`
   - dedicated stream-only instance
   - listens on public `443`
   - reads TLS SNI with `ssl_preread`
   - forwards only approved X/OpenAI suffixes to the real upstream hostname:443
   - sends unmatched SNI to `127.0.0.1:8443`

Outbound handling:

- `iptables nat OUTPUT` matches only user `xproxy`
- matched TCP egress is redirected to `127.0.0.1:7892`
- `mihomo` receives that traffic on `redir-port`
- `mihomo` sends it through `X-OPENAI`

## Subscription Handling

The rollout uses the provided DaWang subscription URL and stores it in:

- `/etc/mihomo/subscription.url`

The config refresh helper is:

- `/usr/local/sbin/update-mihomo-config.sh`

That script:

1. downloads the subscription with `curl -A 'clash.meta'`
2. rewrites the base config into `/etc/mihomo/config.yaml`
3. forces:
   - `allow-lan: false`
   - `external-controller: 127.0.0.1:9090`
   - `redir-port: 7892`
4. injects:
   - `X-OPENAI-AUTO`
   - `X-OPENAI`
   - X/OpenAI domain rules
5. keeps timestamped backups under `/etc/mihomo/backups`

## Key Design Detail: Separate `xproxy` User

This host does **not** use owner-based NAT capture on `www-data`.

Instead:

- stream ingress runs as user `xproxy`
- iptables captures only `xproxy`
- future main-site `nginx` traffic is left alone

This is the main architectural improvement over the old host.

## Listener State After Rollout

Expected steady-state listeners:

- `0.0.0.0:80` -> main `nginx`
- `0.0.0.0:443` -> `nginx-xproxy`
- `127.0.0.1:8443` -> main `nginx` placeholder or future HTTPS site
- `127.0.0.1:7891` -> `mihomo` SOCKS5
- `127.0.0.1:7892` -> `mihomo` redir-port
- `127.0.0.1:9090` -> `mihomo` controller

## Commands For Daily Operations

### Check services

```bash
sudo systemctl status mihomo nginx nginx-xproxy xproxy-iptables --no-pager
```

### Check selected node

```bash
curl -s http://127.0.0.1:9090/proxies/X-OPENAI
```

### Force `X-OPENAI` to a specific node

```bash
curl -X PUT http://127.0.0.1:9090/proxies/X-OPENAI \
  -H 'Content-Type: application/json' \
  -d '{"name":"DaWang-US-Xr2"}'
```

### Refresh subscription config

```bash
sudo /usr/local/sbin/update-mihomo-config.sh
sudo /usr/local/bin/mihomo -t -f /etc/mihomo/config.yaml
sudo systemctl restart mihomo
```

### Validate front-door traffic

```bash
curl -m 20 -I --connect-to x.com:443:127.0.0.1:443 https://x.com/
curl -m 20 -I --connect-to chatgpt.com:443:127.0.0.1:443 https://chatgpt.com/
curl -m 20 -I --connect-to platform.openai.com:443:127.0.0.1:443 https://platform.openai.com/
```

## Rollback Order

Fast rollback:

1. stop `nginx-xproxy`
2. stop `xproxy-iptables`
3. if needed, stop `mihomo`
4. keep main `nginx` running on `80` / `127.0.0.1:8443`

Commands:

```bash
sudo systemctl stop nginx-xproxy
sudo systemctl stop xproxy-iptables
sudo systemctl stop mihomo
```

To remove the transparent redirect only:

```bash
sudo /usr/local/sbin/xproxy-iptables.sh stop
```

## Future Site Hosting Guidance

If this host later needs to serve a real HTTPS site:

1. keep `nginx-xproxy` on public `443`
2. configure the real site in main `nginx` on `127.0.0.1:8443`
3. use valid site certificates there
4. unmatched SNI will then fall back to the real local site instead of the placeholder

This is why the placeholder `8443` listener exists now.
